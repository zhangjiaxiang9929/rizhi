# BTC合约交易机器人修复版本总结

## 🎯 修复的核心问题

### 1. 盈亏计算错误（关键修复）
**原问题**: 盈亏计算未包含手续费，导致盈利虚高
**修复**: 
```javascript
// 正确计算净盈亏
const fee = this.pos.notional * 0.0005 * 2; // 开仓+平仓手续费
const grossPnl = this.pos.notional * pPct / 100;
const netPnl = grossPnl - fee;
```
**影响**: 所有盈亏显示为真实净盈亏，避免误导

### 2. AI顾问未被调用（关键修复）
**原问题**: `engine.js`中未调用`aiAdvise`函数，AI功能完全未用
**修复**: 
```javascript
// 在开仓条件中添加AI分析
const aiResult = await aiAdvise({
  sig, price, rsi: cur15m.rsi, ef: cur15m.ef, es: cur15m.es, em: cur15m.em,
  atr: cur15m.atr, trend: trend1h, score,
  recentCandles: raw15m.slice(-20)
}, this.log.bind(this));
```
**效果**: AI分析现在实际参与决策，置信度≥60%才开仓

### 3. 趋势判断过于简单
**原问题**: 只用一根K线的EMA比较，频繁切换趋势
**修复**: 
```javascript
function getTrend1h(cs1h) {
  if (cs1h.length < 5) return "HOLD";
  const last5 = cs1h.slice(-5);
  const bullishCount = last5.filter(c => c.ef > c.es && c.close > c.em).length;
  const bearishCount = last5.filter(c => c.ef < c.es && c.close < c.em).length;
  if (bullishCount >= 3) return "LONG";
  if (bearishCount >= 3) return "SHORT";
  return "HOLD";
}
```
**效果**: 趋势判断更稳定，减少噪音干扰

### 4. 市场状态识别缺失
**原问题**: 没有区分震荡市和趋势市，在震荡市过度交易
**修复**: 
```javascript
function detectMarketState(cs15m) {
  // 检测振幅和ATR识别市场状态
  if (rangePct < 1.5 && atrPct < 0.3) return "consolidation";
  if (rangePct > 3 && atrPct > 0.5) return "trending";
  return "normal";
}
```
**策略**: 震荡市提高开仓门槛（评分≥70），趋势市正常交易

### 5. 连亏减仓过于激进
**原问题**: 指数级减仓（2^n），连续亏损3次仓位降至12.5%
**修复**: 
```javascript
const reductionFactor = Math.max(0.3, 1 - (lossCount * 0.3)); // 每次减30%
posRatio = Math.max(10, posRatio * reductionFactor); // 最低保持10%
```
**效果**: 保留基础仓位，避免仓位过快降至零

### 6. 冷却期机制不合理
**原问题**: 固定30分钟冷却，不适合不同亏损幅度
**修复**: 
```javascript
let coolSlots = 2; // 默认30分钟
if (lossPct > 2) coolSlots = 3; // 亏损>2%，45分钟
if (lossPct > 3) coolSlots = 4; // 亏损>3%，60分钟
if (lossPct > 5) coolSlots = 6; // 亏损>5%，90分钟
```
**效果**: 根据亏损幅度动态调整冷却时间

### 7. 盈亏比验证缺失
**原问题**: 未验证止损止盈的盈亏比
**修复**: 
```javascript
const riskRewardRatio = tpDist / slDist;
if (riskRewardRatio < 1.5) {
  this.log("warn", `⚠️ 盈亏比不足 ${riskRewardRatio.toFixed(1)}:1，需要至少1.5:1`);
  return;
}
```
**效果**: 确保每笔交易有足够盈利空间

### 8. 保证金检查缺失
**原问题**: 未预留手续费导致资金不足
**修复**: 
```javascript
const feeReserve = notional * 0.001; // 手续费预留（0.1%）
if (actualMargin + feeReserve > margin) {
  this.log("error", `💰 资金不足！需要 ${(actualMargin+feeReserve).toFixed(2)}U，可用 ${margin.toFixed(2)}U`);
  return;
}
```
**效果**: 避免下单失败

### 9. 每日盈亏限制缺失
**原问题**: 没有每日盈亏上限/下限
**修复**: 
```javascript
const dailyTpLimit = this.cfg.dailyTpPct || 5;
const dailySlLimit = this.cfg.dailySlPct || 3;
if (dailyTpLimit > 0 && dailyPct >= dailyTpLimit) {
  this.log("win", `🎯 今日盈利已达${dailyPct.toFixed(2)}%，停止开新仓！`);
  return;
}
```
**效果**: 防止过度贪婪或过度亏损
