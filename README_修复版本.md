# BTC合约交易机器人 - 修复优化版本 v2.1

## 📊 修复的关键问题

### 1. 盈亏计算错误修复
- **问题**: 原版盈亏计算未包含手续费
- **修复**: 添加开仓+平仓手续费（0.05%×2）
- **公式**: `netPnl = grossPnl - fee`
- **影响**: 所有盈亏显示为净盈亏，更真实

### 2. AI顾问未启用修复  
- **问题**: `engine.js`中未调用`aiAdvise`函数
- **修复**: 在开仓条件中添加AI分析调用
- **阈值**: AI置信度≥60%才开仓
- **效果**: 减少无效开仓，提高胜率

### 3. 趋势判断过于简单修复
- **问题**: 只用一根K线的EMA比较，频繁切换
- **修复**: 多K线趋势确认（5根K线中≥3根一致）
- **函数**: `getTrend1h()` 
- **效果**: 趋势判断更稳定，减少噪音

### 4. 市场状态识别缺失修复
- **问题**: 没有区分震荡市和趋势市
- **修复**: 新增`detectMarketState()`函数
- **判断**: 根据振幅和ATR识别市场状态
- **策略**: 震荡市提高开仓门槛（评分≥70）

### 5. 连亏减仓过于激进修复
- **问题**: 指数级减仓（2^n），仓位过快降至零
- **修复**: 线性递减减仓（每次减30%，最低30%）
- **公式**: `reductionFactor = Math.max(0.3, 1 - (lossCount * 0.3))`
- **效果**: 保留基础仓位，避免错过反弹

### 6. 冷却期机制优化
- **问题**: 固定30分钟冷却，不适合不同亏损幅度
- **修复**: 根据亏损幅度动态调整冷却时间
  - 亏损<2%：30分钟
  - 亏损2-3%：45分钟
  - 亏损3-5%：60分钟
  - 亏损>5%：90分钟
- **效果**: 更大亏损需要更长恢复时间

### 7. 盈亏比验证缺失修复
- **问题**: 未验证止损止盈的盈亏比
- **修复**: 计算并验证盈亏比≥1.5:1
- **公式**: `riskRewardRatio = tpDist / slDist`
- **效果**: 确保每笔交易有足够盈利空间

### 8. 保证金检查缺失修复
- **问题**: 未预留手续费导致资金不足
- **修复**: 添加手续费预留检查（0.1%）
- **公式**: `feeReserve = notional * 0.001`
- **效果**: 避免下单失败

## 🚀 如何使用修复版本

### 替换文件步骤：
1. **备份原文件**：
   ```bash
   cp engine.js engine_backup.js
   ```

2. **使用修复版本**：
   ```bash
   cp engine_fixed.js engine.js
   ```

3. **更新依赖**：
   - `ai-advisor.js`保持不变
   - `main.js`需要更新导入（如果有变化）

### 配置参数建议：

**基础参数**：
```javascript
{
  capital: 10000,      // 账户资金（USDT）
  takePct: 3,          // 止盈百分比（建议3%）
  stopPct: 1.5,        // 止损百分比（建议1.5%，盈亏比2:1）
  leverage: 5,         // 杠杆倍数（建议5x）
  posRatio: 20,        // 仓位比例（建议20%）
  
  // 风控参数
  maxLoss: 15,         // 总亏损熔断阈值（15%）
  maxTradesPerDay: 10, // 每日最大交易次数
  reduceAfterLosses: 2, // 连亏2次后减仓
  
  // 每日盈亏限制
  dailyTpPct: 5,       // 每日盈利上限5%
  dailySlPct: 3,       // 每日亏损上限3%
  
  // 动态参数
  trailPct: 1,         // 移动止损百分比（1%）
}
```

**AI配置**：
- 使用默认的千问3 Max模型
- 置信度阈值：60%（可调整）
- 超时时间：8秒
