### 2. BTC/ETH 切换功能修复
- 问题：切换到 ETH 后价格和K线不跟随切换
- 根本原因：
  1. `currentSymbol` 变量不存在
  2. `loadChartKlines` 没传 `symbol` 参数

  - `index.html` 加 `let currentSymbol = "BTC_USDT"`
  - `loadChartKlines` 加 `symbol: currentSymbol`
  - `startPricePoller` 加 `currentSymbol` 参数
  - `main.js` 两个 handler 加 `symbol` 参数支持
  - `engine.js` `fetchKlines(interval, bars, symbol)` 和 `getPrice(symbol)` 加参数
  - 加 `function switchContract(contract) { switchSymbol(contract); }` 别名

### 34. 盈亏算法问题发现与修复（19:52-20:03 UTC）

**问题发现：**
- 用户交易数据显示止盈笔 +9.62U，但手动验算应为约+48U
- 差了约5倍 = 杠杆倍数
- 根本原因：代码中 `pnl = this.pos.notional * pPct / 100 / this.cfg.leverage` 多除了一次杠杆
  - `notional` = 合约数 × 价格 × 0.0001（已是名义价值）
  - 名义价值本身就包含杠杆效应，不需要再除以杠杆

**手续费问题：**
- 开仓0.05% + 平仓0.05% = 双边0.1%
- 以 867张、价格69145.3 为例：notional = 5994.9U，手续费 = 5.99U
- 手续费对高频小盈利影响很大（约6U/笔）

**正确公式：**
```
fee = notional * 0.001  （双边 0.1%）
pnl = notional * pPct / 100 - fee
```

**需要修改的5处：**
1. `startPriceTicker` 实时止损止盈：旧 `/ this.cfg.leverage` → 新 `- fee`
2. `run()` AI巡检全平：同上
3. `run()` 普通止损止盈：同上，`netPnl = pnl`（已含手续费，不再单独减）
4. `partialClose` 手动平仓：`this.pos.notional * ratio * pPct / 100 - fee`
5. `run()` AI减仓 emit pnl：`this.pos.notional * 0.5 * pPct / 100 - this.pos.notional * 0.5 * 0.001`

**当前状态：** 修复方案已告知用户，等待用户应用后验算确认
