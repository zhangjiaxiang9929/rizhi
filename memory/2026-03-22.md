#### 发现的潜在问题（本次审查未修复，仅记录）

1. **`run()` 开仓段有孤立else分支引用未定义变量**
   ```javascript
   } else if (orderBook && orderBook.topBid && orderBook.topAsk) {
     if (sig === "LONG") { sl = ...; tp = atrTP; }  // atrTP 未定义！
   } else { sl = atrSL; tp = atrTP; }              // atrSL/atrTP 未定义！
   ```
   - 这段代码在 `aiResult.allow` 的 else 分支，正常流程中：
     - AI返回skip时 `return`，不会走到这里
     - AI返回allow时执行止损计算后不走else
   - **实际上这段代码永远不会执行**（dead code），但存在语法风险
   - 建议：删除这整段 else if / else 兜底

2. **`runAiDriven()` 里 balance emit 字段不完整**
   ```javascript
   this.emit("balance", {
     balance: this.simAvail,
     start: this.cfg.dryRun ? this.simAvail : this.startBal,
     total: this.simBalance,
     margin: this.simMargin,
     pnl: this.simTotalPnl   // ← 这里传的是totalPnl，但字段名叫pnl！
   });
   ```
   - 应该是 `pnl: 0, totalPnl: this.simTotalPnl, simBalance: this.simBalance`
   - 导致AI主导模式开仓后本次盈亏和累计盈亏显示错误

   - 它们是作为 `dataset.price` 的存储容器（`el.dataset.price = data.sl`）
   - 只在图表画线时读取：`parseFloat(slEl?.dataset.price ?? slEl?.textContent)`
   - 因为用了可选链 `?.`，不会报错，但如果元素不存在则图表止损线不会画出
   - 建议：在 HTML 里加这三个隐藏 span：

     <span id="pos-entry" style="display:none"></span>
     <span id="pos-sl" style="display:none"></span>
     <span id="pos-tp" style="display:none"></span>
