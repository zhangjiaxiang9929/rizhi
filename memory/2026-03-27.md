## BTC合约机器人 - engine.js 根本原因修复 (v10→v11)

### sl/margin is not defined - 真正根本原因
- **表象**: `策略运行异常(非网络): sl is not defined` / `margin is not defined`
- **根本原因**: `run()` 函数中 `sl`, `tp`, `margin`, `notional`, `contracts`, `slPct`, `tpPct`, `fixedSL`, `fixedTP` 全部用 `const`/`let` 声明在 `if (sig !== "HOLD" && score >= 45)` 块内部，当括号错位或异步执行时，这些变量在块外部引用会触发 ReferenceError
- **额外问题**: 多次修改导致括号错位，`if(sig!="HOLD"&&score>=45)` 块提前关闭（多余的 `}`），导致 `dryRun` 开仓代码泄露到块外
- **修复 (v11)**: 在 `run()` 函数顶部统一声明所有变量 (`let sl, tp, margin, notional, contracts, slPct, tpPct, fixedSL, fixedTP;`)，内部 `const`/`let` 改为直接赋值；修复多余括号
- **语法验证**: `node --check engine.js` 通过 ✅
- **上传时间**: 2026-03-27T00:09:07Z (v11)

### 关键教训
- 在同一个 JS 函数中，多次补丁导致括号失衡是危险的 - 每次改动后必须 `node --check` 验证
- `const` 在块内声明 + 多次补丁容易产生 TDZ 或 ReferenceError，应在函数顶部用 `let` 预声明所有共享变量
- 下次修改 engine.js 前先做 `node --check` 确认语法正确再改
