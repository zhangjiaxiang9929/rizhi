## 16:00 UTC 会话补充 — 持仓进度条修复 + 热更新实现

### 持仓进度条修复（最终方案）
**问题根本原因：** 进度条逻辑用 `leftPrice/rightPrice`（按价格大小排列），但箭头应以开仓价为中心参考。当做多盈利，当前价 > 开仓价，但箭头还是显示在开仓价左边，因为开仓价竖线固定在50%而箭头pct计算用的是绝对价格位置。

**最终修复方案（新逻辑）：**
- 开仓价固定在50%（中间）
- 做多：价格 > 开仓价 → 箭头往右移（盈利方向），价格 < 开仓价 → 箭头往左移（亏损方向）
- 做空：方向相反
- slDist = |entry - initSL|，tpDist = |initTP - entry|
- LONG: pct = price >= entry ? 50 + ((price-entry)/tpDist)*50 : 50 - ((entry-price)/slDist)*50
- SHORT: pct = price <= entry ? 50 + ((entry-price)/tpDist)*50 : 50 - ((price-entry)/slDist)*50
- pct = Math.max(1, Math.min(99, pct))

**修复位置：** index.html `updatePositionProgress` 函数内，替换原来的 leftPrice/rightPrice/range/pct计算段

**状态：** 用户确认"现在对了" ✅
